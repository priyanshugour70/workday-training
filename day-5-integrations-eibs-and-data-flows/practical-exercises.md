### Day 5 – Practical Exercises

#### Exercise 1 – Draw Integration Diagrams

- For each of these flows, draw a simple diagram:
  - Workday → Bank (payments, statements).
  - Vendor system → Workday (e.g. expense tool).
  - Workday → Data Warehouse → Power BI.
- For each arrow, note:
  - Data type (e.g. payments, GL balances).
  - Integration type (file/API/EIB, if known conceptually).

#### Exercise 2 – EIB Use Case Brainstorm

- List 5 scenarios where an **EIB** might be used (e.g. journal loads, bulk supplier updates).
- For each, note:
  - Source of the file.
  - Validation steps you would want before import.

#### Exercise 3 – Data Quality Investigation Story

- Imagine a scenario: totals in Power BI do not match a Workday report.
- Write a short **investigation plan**:
  - What would you compare?
  - Which teams would you contact (Controllership, IT, vendor)?
  - How would you document the issue and remediation?
