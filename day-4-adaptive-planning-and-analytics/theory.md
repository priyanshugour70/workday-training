### Day 4 – Theory: Adaptive, SQL & Power BI

#### 1. Adaptive Planning (Conceptual)

- **Purpose**
  - Planning, budgeting, and forecasting.
  - Scenario analysis (e.g. best case, worst case).
- Core ideas:
  - **Models**: structures representing accounts, time, organizations.
  - **Versions**: current budget, forecast, actuals.
  - **Sheets**: where users input or review data.
- Integration with Workday:
  - Uses **actuals** from Workday Financials.
  - Sends back plans/budgets for comparison and reporting.

#### 2. SQL Fundamentals (Relevant to JD)

- Understand:
  - `SELECT` specific columns from a table.
  - `WHERE` to filter rows.
  - `ORDER BY` to sort results.
  - `GROUP BY` with aggregates like `SUM`, `COUNT`.
- In this role:
  - SQL is used to **investigate data issues**, **validate reports**, and support **ad‑hoc analysis**.

#### 3. Power BI Basics for Finance

- Concepts:
  - **Data source connections** (files, databases).
  - **Data model** (tables and relationships).
  - **Visuals** (tables, bar charts, line charts, slicers).
  - **Measures** (simple DAX like sums and ratios).
- For the JD:
  - Use Power BI to **present financial data** clearly.
  - Validate that the data matches **Workday/warehouse outputs**.
